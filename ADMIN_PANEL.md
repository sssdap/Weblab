# 👨‍🏫 Администраторская Панель (Учительская)

## Описание

Админ панель предназначена для преподавателей. Здесь учителя могут:
- Проверять работы студентов
- Смотреть список учеников
- Управлять курсами
- Выставлять оценки

## Логика входа

### Ролевая система

Каждый пользователь имеет роль:
- **`student`** - ученик (по умолчанию при регистрации)
- **`admin`** - преподаватель/администратор

### Вход преподавателя

1. Преподаватель переходит на `/auth/login/admin`
2. Вводит свой email и пароль
3. Система проверяет:
   - ✅ Пользователь существует в Firebase
   - ✅ Пароль верный
   - ✅ **Роль пользователя = `admin`**
4. Если роль не админ - показывается ошибка: *"У этого аккаунта нет доступа к админ панели"*
5. Если все хорошо - редирект на `/admin/dashboard`

## Как назначить администратора

### Способ 1: Используя CLI скрипт (РЕКОМЕНДУЕТСЯ)

```bash
# Перейди в папку проекта и запусти:
npx ts-node scripts/make-admin.ts teacher@example.com
```

**Пример:**
```bash
npx ts-node scripts/make-admin.ts ivan.petrov@school.com
```

**Вывод:**
```
🔍 Ищу пользователя с email: ivan.petrov@school.com
✅ Найден пользователь:
   ID: abc123xyz
   Email: ivan.petrov@school.com
   Имя: Иван Петров
   Текущая роль: student

⏳ Обновляю роль на администратора...

✅ Успешно! Пользователь ivan.petrov@school.com теперь администратор

💡 Он может войти через: /auth/login/admin
```

### Способ 2: Через API endpoint

```bash
curl -X POST http://localhost:3000/api/admin/make-admin \
  -H "Content-Type: application/json" \
  -d '{
    "userId": "user_id_here",
    "adminToken": "your-admin-token"
  }'
```

**Требования:**
- Нужно знать ID пользователя (находится в Firebase Console)
- Нужен специальный `ADMIN_TOKEN` (должен совпадать с `process.env.ADMIN_TOKEN`)

### Способ 3: Через Firebase Console (вручную)

1. Перейди в [Firebase Console](https://console.firebase.google.com)
2. Выбери свой проект
3. Перейди в **Firestore Database**
4. Найди коллекцию `users`
5. Откройте документ пользователя
6. Измени поле `role` с `"student"` на `"admin"`
7. Сохрани

## Проверка доступа преподавателя

После присвоения роли админ, преподаватель должен:

1. Перейти на `/auth/login/admin`
2. Ввести свой email и пароль (те же, что при регистрации)
3. Если всё правильно - попадёт на `/admin/dashboard`

## Структура данных в Firestore

Документ пользователя-администратора выглядит так:

```json
{
  "id": "user123",
  "email": "teacher@example.com",
  "name": "Иван Петров",
  "role": "admin",  // ← Это важно!
  "avatar": "https://...",
  "createdAt": "Timestamp"
}
```

## Тестирование

### Сценарий тестирования

1. **Регистрация нового преподавателя**
   ```
   Email: teacher@test.com
   Пароль: password123
   Имя: Test Teacher
   ```

2. **Назначение администратора**
   ```bash
   npx ts-node scripts/make-admin.ts teacher@test.com
   ```

3. **Вход на админ панель**
   - Перейди на `/auth/login/admin`
   - Введи: `teacher@test.com` / `password123`
   - Должен редиректить на `/admin/dashboard`

4. **Проверка роли**
   - Посмотри на пользователя в Firebase Console
   - Убедись, что `role = "admin"`

## Ошибки

### "У этого аккаунта нет доступа к админ панели"

Это значит, что пользователь существует и пароль верный, но его роль не администратор.

**Решение:**
```bash
npx ts-node scripts/make-admin.ts user@example.com
```

### "Ошибка входа"

Проверь:
- ✅ Email и пароль верны
- ✅ Пользователь зарегистрирован
- ✅ Email совпадает в точности (заглавные буквы)

## Архитектура логина администратора

```
┌─────────────────────────────────────┐
│  /auth/login/admin (AdminLoginForm) │
└────────────┬────────────────────────┘
             │
             ├─ Валидация email/пароль
             │
             ├─► useAuth().signIn()
             │   └─► Firebase аутентификация
             │   └─► Создание session cookie
             │
             ├─ Проверка role === 'admin'
             │
             └─► /admin/dashboard ✅
                 (если админ)
                 
             или
             
             └─ Ошибка: "нет доступа" ❌
                (если не админ)
```
